Relationships carry behavior.

Link, Group, and Symbol models hold persistent content. Their update and deletion rules determine what can coexist and what happens when records change.

Invariant map

Rules that keep records consistent

These rules execute during model updates or persistence operations, not only as presentation behavior.

H

Hidden excludes groups

updateLink() sets isHidden = true and replaces groups with an empty array. Hidden and regular group membership cannot coexist after this update path.

G

Group deletion preserves links

The Link-to-Group relationship uses a nullify delete rule. Removing a group removes its reference from each link without deleting those links.

R

Reminder disabled clears time

When reminders are disabled, the model sets reminderDate = nil and reminderAllDay = false rather than retaining inactive scheduling state.

S

Symbol type has a safe fallback

The enum is stored as a raw string. An unknown value decodes to .icon, keeping the display path valid.

C

Color becomes primitive data

SwiftUI Color is converted into RGBA doubles for persistence and reconstructed when the symbol is rendered.

P

Preferences use a separate store

Hidden-feature availability and title/caption line limits live in UserDefaults rather than SwiftData records.

Record lifecycle

New records are deliberately created outside the context, then inserted only after a valid form submission.

    QUERY -->|"Edit"| UPDATE["Mutate model + context.save"]
    UPDATE --> QUERY
    QUERY -->|"Delete"| DELETE["ModelContext.delete + save"]
