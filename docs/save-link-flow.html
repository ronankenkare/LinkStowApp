    actor User
    participant Main as MainView

    alt Paste action
        User->>Main: Tap Paste
        Main->>Main: Read UIPasteboard text
    else Manual entry
        User->>Main: Type URL and tap Search
    end

    Main->>Ctrl: verifyLink(input)
    Ctrl->>Meta: verifyLink(input)
    Meta->>Web: Request original URL

    alt Original is reachable
        Web-->>Meta: HTTP 200...399 + parseable HTML
        Meta-->>Main: Verified original URL
    else Input has no HTTP scheme
        Meta->>Web: Request https:// + input
        alt HTTPS is reachable
            Web-->>Meta: Valid response
            Meta-->>Main: Normalized HTTPS URL
        else HTTPS fails
            Meta->>Web: Request http:// + input
            Web-->>Meta: Valid response or failure
            Meta-->>Main: Normalized HTTP URL or nil
        end
    end

    alt URL verified
        Main->>Ctrl: fetchWebsiteTitle(URL)
        Ctrl->>Meta: parse title
        Meta->>Web: Download page HTML
        Web-->>Meta: HTML response
        Meta-->>Main: title or nil

        Main->>Ctrl: fetchWebsiteDescription(URL)
        Ctrl->>Meta: parse meta description
        Meta->>Web: Download page HTML
        Web-->>Meta: HTML response
        Meta-->>Main: description or nil

        Main->>Ctrl: fetchWebsiteIcon(URL)
        Ctrl->>Meta: find + download image
        Meta->>Web: Download page and candidate icon
        Web-->>Meta: HTML + image bytes
        Meta-->>Main: favicon data or nil
        Main->>Main: state = addLink
    else Verification failed or threw
        Main->>Main: state = linkNotFound
        User->>Main: Add Link as Entered
    end

    User->>Main: Tap Add
    Main->>Ctrl: createLink(URL, metadata)
    Ctrl-->>Main: Unsaved LinkModel
    Main->>Main: Reset entry state
    Main->>Editor: Present sheet with draft
    Editor->>VM: initialize(from draft, isNew)
    User->>Editor: Edit fields, symbol, groups, visibility

    opt User marks link hidden
        VM->>VM: authenticateUser()
        VM-->>Editor: Set isHidden only on success
    end

    User->>Editor: Tap Save
    Editor->>VM: save(draft, isNew)
    VM->>Ctrl: addLink or updateLink
    Ctrl->>Model: updateLink(all edited values)
    alt Hidden
        Model->>Model: isHidden = true, groups = empty
    else Visible
        Model->>Model: isHidden = false, groups = selected groups
    end
    alt New link
        Ctrl->>Data: insert(LinkModel)
    else Existing link
        Ctrl->>Data: save ModelContext
    end
    Data-->>Main: @Query publishes updated list

Nested fallback

How icon enrichment searches

The favicon operation has its own ordered decision chain after the page HTML has been retrieved.

flowchart LR
    HTML["Parse page HTML"] --> REL["First rel~=icon candidate"]
    REL -->|"download succeeds"| DONE["Return image bytes"]
    REL -->|"missing or invalid"| APPLE["Apple touch icon candidates"]
    APPLE -->|"download succeeds"| DONE
    APPLE -->|"missing or invalid"| MASK["Mask icon candidate"]
    MASK -->|"download succeeds"| DONE
    MASK -->|"missing or invalid"| OG["Open Graph image"]
    OG -->|"download succeeds"| DONE
    OG -->|"missing or invalid"| ROOT["/favicon.ico at site root"]
    ROOT -->|"valid image or small blob"| DONE
    ROOT -->|"failure"| NONE["Return nil; use default link icon"]
1

Verification is tolerant

A user can continue after verification failure. The draft is created from the literal input instead of losing the workflow.

2

Metadata is optional

Title, description, and icon return empty or nil independently. Each missing field has a UI fallback.

3

Saving is explicit

The model remains outside ModelContext while the editor is open, so cancelling a new-link sheet leaves no persisted record.

As-built network characteristic: verification, title, description, and icon retrieval each request the page separately. A successful save path can download the same HTML at least four times before any icon-image request.
