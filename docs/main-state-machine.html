05 · Behavioral state view

The home screen runs two state systems.

The primary URL control manages capture and draft creation. A separate filter selector chooses visible, grouped, or authenticated hidden records.

MainButtonStateFilterStateOne dormant state

Primary action state machine

Transitions change the button label, button color, keyboard meaning, and action handler. The editor draft is shown as an outcome outside the enum.

Filter selection state machine

Only one selection type is active at a time. Selecting the active bubble again returns to the unfiltered visible-link list.

State outputs

What each state changes

The state enums do not exist in isolation; they select data sources, visuals, and allowed operations.

A

Action semantics

MainButtonState selects Paste, Search, Add as Entered, Add, or Filter behavior and its matching label/color.

D

Data source

Hidden selection switches from the visible SwiftData query to the hidden query. A group selection then filters by identity membership.

V

Visible controls

When a filter is selected, unrelated group bubbles collapse and the sort-direction bubble appears beside the active choice.

Reachability gap: .filterLink is fully represented in the UI and in filteredLinks, but no source path calls changeMainState(to: .filterLink). Text filtering is therefore dormant in the current build.
